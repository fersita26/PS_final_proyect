API Documentation

Available Endpoints

1. Get All Tasks

URL: /api/tasks

Method: GET

Description: Retrieves all tasks for the authenticated user.

Response:

{
    [
        {
            "id": 1,
            "content": "Task 1",
            "deadline": "2024-12-25"
        }
    ]
}

2. Create a New Task

URL: /api/tasks

Method: POST

Description: Creates a new task for the authenticated user.

Request Body:

{
    "content": "New Task",
    "deadline": "2024-12-20"
}

Response:

{
    "id": 2,
    "content": "New Task",
    "deadline": "2024-12-20"
}

3. Update an Existing Task

URL: /api/tasks/<task_id>

Method: PUT

Description: Updates an existing task.

Request Body:

{
    "content": "Updated Task",
    "deadline": "2024-12-22"
}

Response:

{
    "id": 2,
    "content": "Updated Task",
    "deadline": "2024-12-22"
}

4. Delete a Task

URL: /api/tasks/<task_id>

Method: DELETE

Description: Deletes a task.

Response:

{
    "message": "Task deleted successfully"
}

5. Query Tasks by Date

URL: /api/tasks/by_date?date=YYYY-MM-DD

Method: GET

Description: Retrieves all tasks for a specific date.

Response:

{
    "tasks": [
        {
            "id": 1,
            "content": "Task for today",
            "deadline": "2024-12-20"
        }
    ]
}

For more details, please contact the developer.
